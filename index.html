Are You Even Turing Complete?

If you know who Alan Turing was and what Turing-completeness implies, my guess is your first thought upon reading the question is: yes. I am going to try and convince you that the answer is not clear at all. At least not with what is currently known about the human mind. But first, it is necessary to go over the basics of theoretical computer science. So, let's reason about the question step by step.

Computation

Although no universally accepted definition of computation exists, I think virtually everyone understands the essence shared across many attempts to formalize one. The notion most people have is that computation is a well-defined series of unambiguous instructions to be executed in a specific order. For practical purposes, let's refer to such a series of instructions as program.

Disagreements often appear when a formal statement is proposed. Presently, the most popular formalization is that a computation is any statement that can be expressed in the initialization parameters of a Turing machine. The set of all possible Turing machines constitutes a computational model, typically known simply as the Turing machine. This generates ambiguity because the same term is also used to refer to a single Turing machine. To avoid this, I write Turing machine when referring to the computational model.

It is worth mentioning any notion of computability eludes the aforementioned definition. Whether a program can be realized as a Turing machine has nothing to do with whether it can be effectively computed at all because Turing machines exist for programs that never halt (i.e., never finish). A trivial example of this is a while loop without a falsifiable stopping condition. This simple program would provably continue forever, breaking one of the requisites to be considered an effectively computable function. Despite not being effectively computable, this program can be realized as a Turing machine that loops forever. Effectively computable functions are the focus of the Church-Turing thesis because they work with algorithms. The definition of program I am using here is much weaker.

Turing Machine

Any theory stemming from the definition of computation introduced in the previous section would assume that, for every program, there exists at least one Turing machine that realizes it. Whether this is true or false, is currently unknown. Although it is generally accepted to be true, because all available evidence suggests it is true. Let's go over the evidence.

Besides the Turing machine, there exist many alternative models of computation. Yet, not all models are equal. Some are strictly less powerful than others in terms of the programs they can realize. In general, given two computational models M and M, if the representational power of M is greater than or equal to the representational power of M, then it is possible to translate any realization of a program in M to a realization in M. This translation is bi-directional if and only if M and M can realize the same set of programs. At the time of writing this, no computational model is known that contains programs that are unrealizable by the Turing machine. All known models are either equivalent or inferior to the Turing machine in this respect.

Examples of computational models as powerful as the Turing machine are:

  • Lambda calculus

  • (Some) Cellular automata

  • Register machines

Examples of computational models strictly less powerful than the Turing machine are:

  • Finite-state machines

  • Pushdown automata

  • Combinational logic circuits

When a computational model can realize the same set of programs as the Turing machine, it is said that the computational model is Turing-complete. Turing completeness is a statement about computational models. When I say register machines are Turing-complete, I do not mean some machines happen to be Turing-complete. That sentence would not make sense, because Turing-completeness is a comparison between sets, not a property of the elements within. Yet, the term is often used when referring to a single machine that can simulate any Turing machine.

I believe the source of the misconception is the way Turing completeness is often (if not always) proved. Between 1936 and 1937, Alan Turing introduced the universal Turing machine. Unlike typical Turing machines which can do only one thing, the universal Turing machine is programmable. This enables it to simulate any other Turing machine, be it programmable or not. To prove another computational model is Turing-complete, it is sufficient to show it can realize the universal Turing machine.

When referring to a computational model, the correct term to use is Turing-completeness. When referring to the realization of a program, the correct term to use is universal. A universal program can simulate any Turing machine. By extension, the computational model to which that program's realization belongs is also Turing-complete.

Turing-Complete

Given a Turing-complete model, how could it be constrained such that it stops being Turing-complete?

Looking at programs within a computational model, one can generally understand them as automata navigating state spaces. Whatever constraint one could think of, if it renders the state space for all programs finite, then the computational model stops being Turing-complete. For example, restricting the maximum number of time steps essentially restricts the maximum path length, thus shrinking the state space to a finite subset within reach from the initial state. However, just because some programs in a computational model can navigate infinite state spaces does not mean the model is Turing-complete. Many elementary cellular automata can navigate infinite state spaces, but most are not Turing-complete. In retrospect, this explains why Turing completeness proofs almost always involve the universal Turing machine. Articulating a sufficient condition to enable Turing completeness is not as easy as articulating a sufficient condition to break it.

Are You Even Turing-Complete?

Interpreting the question literally, the answer is no. But the intended meaning is: Are you even universal?

Well, Are You?

Given how complex humans are, it is only natural to think our minds are universal. Surely, if given enough time, ink, and paper a human could simulate any Turing machine, right? I think if we should learn something from large language models (LLMs), it is that complexity does not automatically imply universality. Despite some papers claiming transformer architectures are Turing-complete, the truth is that there are languages they simply cannot handle. Dr. Hessameddin Akhlaghpour wrote a blog post, Transformers Arent Turing-complete, But a Good Disguise Is All You Need, in which he explains some of the logical fallacies used to support arguments for transformers' Turing-completeness. Furthermore, there is a growing body of work tacitly studying their limitations. A notable example is: Your Transformer May Not be as Powerful as You Expect.

When we say the human mind is Turing-complete universal, we often base that belief on unfounded human-centrism. In my opinion, we have no evidence to conclude human minds are universal. Even when we exhibit behaviors that suggest so because universality could emerge from interactions between non-universal agents and their environment.

Consider Conway's Life variant HighLife. The game is played on an unbounded grid of square cells, each of which can be in one of two possible states: alive or dead. The state of each cell at the next time step is determined by a set of simple rules based on the number of live neighbors. Each cell's receptive field is fixed in size (33), meaning if one were to enumerate all the possible things a single cell could perceive, the result would be a finite list (233). So, a single cell can be effectively realized by a look-up table, one of (if not the) least powerful computational models in the hierarchy. Despite this stark limitation, if one arranges a few live cells on a board of unbounded size and sets no time limit, eventually some patterns might emerge that exhibit complex behavior. Some can even replicate themselves. Furthermore, if arranged properly, these cellular automata can even simulate any Turing machine.

I argue that human's seemingly universal computation capabilities could be something that emerges from our interaction with the environment, instead of something that happens in isolation within our minds.

So, going back to the original premise: Are you even Turing-complete universal? I guess the correct answer should be I do not know. Right now there is no evidence for or against either possibility.

(In my humble opinion.)

If I managed to convince you that you might not be Turing-complete universal, then I think you fell for a logical fallacy. In my argument, the analogy to cellular automata is set up such that one can immediately make the connections cellperson and the boardenvironment. While not necessarily wrong, it reduces the discussion to a problem of definitions. A case of "tomayto, tomahto". Let's see why.

Observe a classical Turing machine has one tape and one head. Applying the same analogy, the head would be an agent powered by a look-up table. The tape would be the environment in which this simple agent makes decisions and takes action. So, by the same logic: headcell and tapeboard. Then we could conclude that the head of a Turing machine is not Turing-complete universal in isolation. You see? While not wrong, it is also trivial. Of course, the head of a Turing machine is not universal by itself! That's the whole point of a Turing machine!
